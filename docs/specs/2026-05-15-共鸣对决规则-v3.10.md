## 一、资源系统

| 资源 | 范围 | 恢复机制 | 含义 |
|---|---|---|---|
| **心跳**（HP）| 0-15 | 不恢复 | Memoria 的存在度。归零 = 消散 |
| **Sync 能量** | 0-3 | 每回合自动 +1 | 用于"凝固"（-2）和"散频"（-1）|

## 二、6 个动作

### 攻击类

#### 触达（普攻）· 快动作
- 消耗：无
- 伤害：`attackBase + min(combo, comboMax)`
- 连击 buff：连续触达伤害递增（默认 2/3/4，A 角色 3/4/5）
- 连击归零条件：自己出过非触达动作 **或** 被对方反伤

#### 爆响（强攻）· 慢动作
- 消耗：1 到 N 点共鸣值（玩家自选，N = 当前共鸣值）
- 伤害：`1 + 消耗的共鸣值`（最高 11）
- **可分配机制**：蓄到 6 点不一定全消耗——小爆响（1-2 点）骗对方花资源，留点数下回合大爆响
- 被触达打断：消耗 < 5 点时被打断，强攻方扣 2 心，共鸣值保留
- **🆕 穿透打断**：**消耗 ≥ 5 点（即伤害 ≥ 6）时无法被触达打断**（v3.10）

### 防御类

#### 守护（普防）· 快动作
- 消耗：无
- 效果：挡住触达 + 反伤 `blockReflect`（默认 1，B 角色 2）
- **挡不住爆响**（爆响穿透普防）

#### 凝固（有效防）· 慢动作
- 消耗：**2** Sync 能量（v3.4 改自 1，修支配策略 bug）
- 效果：挡住所有攻击（含爆响）
- **挡触达反伤**：`guardReflect`（默认 1，B 角色 2）
- **🆕 挡爆响反伤**（v3.9）：`guardReflect + ⌊对方消耗蓄能 / 3⌋`
  - 对方消耗 3 点 → 反伤 +1
  - 对方消耗 6 点 → 反伤 +2
  - 对方消耗 9 点 → 反伤 +3
- 对空浪费：对方没攻击时，能量仍消耗 2

### 资源类

#### 聚频（蓄能）· 慢动作
- 消耗：无
- 效果：共鸣值 + N，N 按连蓄回合数递增
  - 默认：1 / 3 / 6 / 10（上限 10）
  - C 角色（chargeBoost: 1）：2 / 4 / 7 / 11（截断上限 10）
- 连蓄归零条件：自己出过非聚频动作 **或** 被爆响打断

#### 散频（破蓄）· 慢动作
- 消耗：1 Sync 能量
- 效果：**仅当对方本回合也选聚频时**——清零对方共鸣值
- 猜错（对方没聚频）→ 能量浪费 + 这回合无攻无防

## 三、克制基础原则（铁律）

### 铁律 1：速度决定打断
快动作（触达 / 守护）打断慢动作（爆响 / 凝固 / 聚频 / 散频）。被打断方的动作无效但**资源保留**。

### 🆕 铁律 1.5：穿透打断（v3.10）
**爆响消耗蓄能 ≥ 5 时无法被触达打断**——双方都打中（互殴）。这是给"all-in 爆响"的真实奖励路径。

### 铁律 2：攻防关系
- 触达 ↔ 守护：攻被反
- 爆响 ↔ 凝固：攻被反（反伤随蓄能递增）+ 凝固消耗 2 能量
- 爆响 ↔ 守护：**爆响穿透**（守护无效）
- 触达 ↔ 凝固：攻被反 + 凝固浪费 2 能量

### 铁律 3：资源动作的代价
聚频和散频本身没有防御——被任何攻击都会受伤。

### 铁律 4：被反伤打断连击
触达被守护/凝固反伤时，自己的连击数清零。

### 铁律 5：聚频被攻击的差异
- 被触达：扣 2 心，**共鸣值仍进帐**（小骚扰）
- 被爆响：扣大血，**共鸣值不进帐 + 连蓄重置**（真正打断）
- 被散频：清零所有已有共鸣值

## 五、特殊机制详解

### 连击 buff（触达专属）

| 第几次连续触达 | 默认伤害 | A 强袭伤害 |
|---|---|---|
| 第 1 次 | 2 | 3 |
| 第 2 次 | 3 | 4 |
| 第 3 次（及之后）| 4（封顶）| 5（封顶）|

### 连蓄递增（聚频专属）

| 第几次连续聚频 | 默认 | C 蓄势 |
|---|---|---|
| 第 1 次 | +1 | +2 |
| 第 2 次 | +3 | +4 |
| 第 3 次 | +6 | +7 |
| 第 4 次 | +10（上限 10）| +11（上限 10）|

### 爆响分配机制（v3.0）
- 玩家自选消耗 1 到当前共鸣值之间任意值
- 伤害 = 1 + 消耗的点数
- 战术应用：
  - 小爆响（2 点 = 3 伤害）→ 骗对方凝固消耗能量
  - 留下蓄能下回合大爆响 → 对方没能量了挡不住
  - **🆕 蓄到 5+ 点穿透打断**（v3.10）—— all-in 终结

### 爆响 vs 爆响（差额制）
- 双方都爆响 → 比伤害高低
- 高方赢，对方扣**差额伤害**
- 双方共鸣值都按各自消耗扣

### 同蓄减半
- 双方都聚频时 → 各自点数减半进帐（向下取整，至少 +1）

### 凝固反震递增（v3.9）
- 凝固挡爆响时，反伤 = `guardReflect + ⌊对方消耗蓄能 / 3⌋`
- 蓄能越大反越疼——给凝固"克 OP 爆响"的真实定位

### 穿透打断（v3.10）
- 爆响消耗蓄能 ≥ 5（即伤害 ≥ 6）→ 触达打不断
- 双方都打中：爆响方扣 `eAtkDmg`（对方普攻），对方扣 `1 + pSpend`（爆响伤害）
- 战略意义：破解"双反制循环"——对方不能再白嫖触达打断大爆响

## 六、胜负判定

- 任一方 **心跳 ≤ 0** → 战斗结束
- 心跳归零方 = 消散（败）
- 另一方 = 共鸣成功（胜）

### 共鸣等级（按胜方剩余心跳分级）

| 胜方剩余心跳 | 共鸣等级 | 叙事 |
|---|---|---|
| > 10 | **Signal · 信号（R）**| 她在远方挥手 |
| 3-10 | **Echo · 回响（SR）**| 她朝你伸出手 |
| < 3 | **Resonance · 共鸣（SSR）**| 险胜——她从光中走出 |

**设计哲学**：越险胜共鸣等级越高——奖励"在危险边缘还能守住"，不奖励碾压式获胜。

## 七、角色差异化（v3.5 启用）

每位 Memoria 的数据结构：

```js
{
  id, letter, name, archetype, desc,
  attackBase,    // 触达基础伤害（默认 2）
  comboMax,      // 连击递增上限（默认 2）
  blockReflect,  // 守护反伤（默认 1）
  guardReflect,  // 凝固反伤（默认 1）
  chargeBoost,   // 连蓄每次额外 +N 点（默认 0）
}
```

### v3.10 三角色配置

| 角色 | 字段调整 | 玩法定位 |
|---|---|---|
| **A · 强袭** | `attackBase: 3` | 触达基础伤害 3——连击 3/4/5，**主动出击型** |
| **B · 格挡** | `blockReflect: 2, guardReflect: 2` | 所有反伤 +1——普防/凝固都反 2，**反击型**（爆响反震基础 = 2 而非 1）|
| **C · 蓄势** | `chargeBoost: 1` | 连蓄递增更快——聚频 2/4/7/11，**爆发型** |

### 平衡数学验证

每角色每回合理论收益 +1 心价值：
- A：每次普攻多打 1 心
- B：每次反伤多反 1 心
- C：每次聚频多 1 共鸣值（≈ 多 1 心潜在伤害）

**三者完全对称**——选哪个都不亏。

## 八、核心理解（4 句话讲清整个游戏）

1. 这是一场**频率较量**——不是物理伤害，是 Keeper 借给 Memoria 频率对抗 Static 杂讯
2. 核心循环 = **聚集力量（聚频）→ 爆发（爆响）↔ 反制（散频/凝固）↔ 节奏控制（触达/守护）**
3. **速度决定打断 + 穿透阈值** + **资源决定深度**（能量 / 共鸣值）= 多层博弈
4. **越险胜共鸣等级越高**——奖励"在危险边缘守住"的玩家

## 十、视觉系统（v3.6 + v3.8）

主战斗的视觉层级（重要 → 装饰）：

### 信息层（必看）
- **HP 大数字**：角色卡角落 36px 数字，HP 低时变金/红 + 脉动
- **伤害飘字**：扣血时角色卡上方飘出 `-2` 红字
- **蓄能数字 + 光柱**：金色光柱可视化 + 数字精确读数
- **能量数字 + 圆环**：圆环视觉化 + `EN 2/3` 精确读数

### 状态层
- **心跳波形**：HP 视觉化（HP 低时跳得急、线变细）
- **角色卡呼吸动画**（持续 3s 循环）
- **HP 危急强化**：< 33% 卡片红色脉动；< 20% 持续抖动
- **连击/连蓄 badge**：状态条边显示

### 氛围层
- 极光背景（深空 + 青/紫/金 4 色）
- 扫描线（细微示波器感）
- 上升粒子（12 个，呼应"瞬间漂浮"）
- 危险 vignette（HP 低时屏幕边缘红色脉动）

### 事件层
- **出招分镜**：4 帧——聚焦 / 喷射 / 撞击 / 结算
- **6 个 SVG 几何符号**（触达=箭头、爆响=八角、守护=双环、凝固=六边形、聚频=上升波、散频=斜十字）

- **破蓄斩切**：紫色斜光横扫
- **胜利特写**：`RESONANCE` 渐变大字
- **失败特写**：`STATIC` 灰色消散

## 十一、音效系统（v3.7 ambient 重做）

设计语言：**空灵 / 共鸣 / 频率感**——避免 chiptune 8-bit 卡通

技术栈：
- Web Audio API 合成（零外部依赖）
- 卷积混响（2.5s）+ master compressor
- 慢 attack（40-150ms）+ 长 release（300ms-1.2s）
- 多层泛音叠加（基频 + 五度 + 八度）
- detune 制造空气感

每个动作的音色（贴合世界观叙事）：

| 事件 | 音色描述 |
|---|---|
| 触达 | 水滴落水面 1320Hz + 涟漪高八度 |
| 爆响 | 80Hz 低频持续 + 多层和声 + 长拖尾（粒子加速器） |
| 守护 | 440-880Hz 能量屏障嗡鸣 |
| 凝固 | 880-2200Hz 多层泛音水晶 |
| 聚频 | 220→660Hz 上升 + 多层八度 |
| 散频 | 1200→60Hz 频率断裂 + 高频噪 |
| 命中 | 60Hz 心跳脉冲低频"咚" |
| 爆响特写 | 55Hz 巨大冲击 + 多层泛音 + 1.4s 余响 |
| 破蓄斩切 | 2200Hz 锐利切割 + 5500Hz 高频噪 |
| 胜利 | 880-2640Hz 高频泛音链渐显（极光闪现）|
| 失败 | 220→55Hz 频率消散 + detune 微失调 |
