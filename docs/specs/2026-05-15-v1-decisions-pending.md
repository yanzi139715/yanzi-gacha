## 二、需用户决策的窗口（按优先级）

### 🟡 D1 — Echo Shard 兑换比例校准
**现况**：100 碎=1 抽券 / 500 碎=频率定位 / 1000 碎=离线原图权。
**问题**：这是拍脑袋的数字，需要做经济模型。
**关键假设**：
- 重复 SSR +50 碎/张，玩家全集需重复 ~20 次才换 1 张定位
- 与 V1 白嫖经济（~36 Beacon/月）对比是否合理
**建议产出**：用户起来后给我"目标月留存付费率 / 全集预算天数"，我重新算。

### 🟡 D2 — Lv.4/5 解锁的实际内容
**现况**：UI 显示"解锁 Lv.5"但没真正解锁什么。
**清单**：
- Lv.1（首抽）✅ 已实现：卡入图鉴
- Lv.2 = 4K 原图下载？需要服务端鉴权（V2.0 才能做）
- Lv.3 = 拍摄花絮视频？妍子需要提供资产
- Lv.4 = 手写感谢卡？需要扫描件
- Lv.5 = 专属语音 / 动态壁纸？妍子需要录音 / 提供素材
**建议产出**：起来后告知哪几档内容已经有素材，先实现这几档；其余留 TODO。

### 🟡 D3 — 频率定位的选卡 UI
**现况**：用 `prompt()` 弹出文本列表（极简）。
**待替换**：精美 modal，类似战斗皮肤选择器那种 grid 卡片。可以复用 `.bs-picker-overlay` 样式。
**自动处理**：先用 prompt，能跑通流程。

### 🟡 D4 — 法律合规：cos IP 与角色名
**核心风险**：抽卡商品名 2B / 雷电将军 / 雅儿贝德 / 胡桃 等都是其他公司 IP（详见复盘）。
**两条路线**：
- **A**：保留角色名（私域不公开运营，IP 方追究难度较低，但 V2.0 接真实支付危险）
- **B**：抽象化重命名（白衣战姬 / 雷之巫女 / 黑翼守护者），失去 cos 识别度但合规
**建议产出**：起来决定走哪条。如果 B，我可以批量改 CHARACTERS.name 和 description。

### 🟡 D5 — 战斗皮肤的 archetype 绑定细节
**现况**：22 个 cos 已分配 A/B/C 型（`BATTLE_PROFILES`）。
**潜在调整**：
- 部分分配偏主观（如 02·花嫁 → C 型 OVERFLOW，是不是该是 B？）
- 妍子作为 coser 整体的"出战型号"现在写死 A 型，是否合理？
**自动处理**：保持现状，可后期调单条。

### 🟡 D6 — Aria（虚拟 Memoria）的接入路径
**现况**：spec/index.html 提到 Aria 是原创虚拟 IP（C 型 OVERFLOW）。
**问题**：Aria 在主游戏 CHARACTERS 列表中**还不存在**，只有 wireframe 提及。
**待补**：要不要在 V1 就接入 Aria 作为第二位 Memoria（原创 IP 不存在版权风险，是商业化优解）？
**建议产出**：起来决定 V1 是否接 Aria，如果接，需要提供 4-14 张原创素材。

### 🟡 D7 — 离线原图下载权（1000 碎兑换）
**现况**：现在弹窗说"V2.0 接后端后启用"，不扣碎片。
**关键依赖**：需要后端 + 鉴权 + CDN，不属于 V1 前端能解决。
**自动处理**：保留兑换入口（让玩家知道终极目标），但 V1 不实际生效。

### 🟢 D8 — 重复抽卡仪式优化
**现况**：连抽 10 张中间点击屏幕可 skip 仪式（已加），但 SSR cut-in 不可 skip。
**待评估**：用户实际玩 100+ 次后会不会嫌烦？需用户测试反馈。
**自动处理**：等用户体验后调。

### 🟢 D9 — 桌面端外侧装饰
**现况**：左右两侧只有粒子背景 + 顶部水印。
**可扩展**：放当期 banner 海报、SSR 收藏成就、好友守护榜单等。
**自动处理**：当前简洁，等内容定了再加。

## 四、给用户的 Quick Test 步骤

起来后想快速验证我做的东西，按此顺序操作：

1. **打开主游戏** `http://localhost:8765/` — 看顶部状态栏多了 ✦ 碎片图标（金色）
2. **设置面板** 右上齿轮 — 看到「快速抽卡」toggle
3. **抽到重复 SSR**（开发者控制台可注入 `state.collection.push('2b_0')` 再抽 2b_0）— 看到卡上"解锁 Lv.X / +50 共鸣碎片"
4. **图鉴 tab** — 滚动看：守护进度 → 关于妍子 → 频率战档案 → 切换主皮肤 → **频率战皮肤三槽位**（A/B/C）→ cos 网格
5. **点战斗皮肤的「切换」按钮** — 弹出 modal 选 SSR
6. **商店 tab** — 看到「共鸣碎片兑换」section 三个兑换包
7. **桌面端打开** — 看虚拟手机框 + 极光粒子 + 顶部水印
8. **进战斗游戏 `battle/`** — 战斗中按键盘 `1` 选触达、`Space` 确认（PC 端快捷键）

## 五、不该误踩的红线

- **不要**在 V1 开启真实支付（现在所有"购买"都是 `confirm()` 模拟）
- **不要**在 V1 上线公域 SEO / 二维码扫码分享（私域起步、绕开 IP 监控）
- **不要**让"下载原图"功能直链原图（V2.0 必须走鉴权链接）
- **不要**修改 `_localSoulId`（用户已绑定的话，会导致云端无法对应）
