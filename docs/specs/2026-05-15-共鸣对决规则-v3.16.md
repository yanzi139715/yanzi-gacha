## 四、签名技能（v3.15 新增）

每角色一个独有 ultimate，**每局只能用一次**。条件满足时 HUD pill 发光提示，玩家点击进入卡片 spotlight 演出（2.2s），落幕后进入 charged 等待生效，下次（或两回合内）触发对应动作即生效。

| 角色 | 技能名 | 触发条件 | 效果 | 生效动作 |
|---|---|---|---|---|
| 妍子 | **烈触达** (BLAZE TOUCH) | 连击 ≥ 3 | 下次触达 +2 伤害 + 穿透守护（凝固反震仍能挡）| attack |
| 小雨 | **共鸣回响** (RESONANCE ECHO) | HP ≤ 6 | 两回合内首次防御反伤命中时回血 = 反伤值 ×150% + 2 | block / guard |
| Aria | **共鸣溢出** (OVERFLOW) | 蓄能 ≥ 8 | 下次爆响穿透守护 + 蓄能不消耗 | power |

### 生效细则
- charged 状态下伤害/回血在 `resolveTurn` 之后覆盖原值
- 烈触达：覆盖伤害为 `attackBase + min(combo, comboMax) + 2`，穿透 block（r.pDmg=0），但 guard 仍能反震
- 共鸣回响：在 reson bonus 之前算 heal（不放大反震谐振的 +bonus）
- 共鸣溢出：保留 spend 但 r.pC 不扣，穿透 block，但 guard 仍能反震

### 卡片 spotlight 演出（v3.16）
- 全屏暗背景 + 角色专属背景动画（A 焰焰光 / B 紫粉镜像 / C 金色极光）
- 角色立绘居中放大（radial mask 融入背景）
- 字幕分层浮现：letter → 技能名 → 英文 → 分割线 → tagline → effect
- 总时长 2.2s，期间 phase=ult 锁，玩家无法 commit

## 五、谐振演出（v3.11/v3.13/v3.14 新增）

### 触达谐振（v3.14）
- 触发：连续 3 次触达成功命中（`r.pCb === 3 && r.eDmg > 0`）
- 演出：橙金三层同心圆扩散 + 6 道箭矢光放射 + 金白 core + 「触 达 谐 振」金字 + 屏幕级金粉 flash
- 仅在 ===3 时演出一次，后续 streak 不重复演出

### 反震谐振（v3.13）
- 跟踪 `defStreak: { block, guard }` 各自计数，连续同种防御成功 +1，换种或失败清零
- 反伤递增：第 N 次成功反伤 +(N-1)，上限 +3
- 第 3 次成功触发演出：蓝-粉-金三层圆波 + 8 道 mirror-shard + 「反 震 谐 振」金字 + 屏幕级金粉 flash

### 普通连击/防御 FX（v3.11）
- 连击 ≥2 命中：极光环 + 频率光柱 + 「频率同步 ×N」（轻量版）
- block 反伤命中：单层六边晶 + 蓝色 + 「镜 像 反 射」
- guard 反伤命中：双层六边晶 + 蓝粉 + 「瞬 间 守 护」

## 六、三档 AI 难度（v3.12）

PrepScreen 三段切换控件，默认中阶。

### 共用工具
- `predictActionDist(subj, foe, subjH, foeH, recentSubjActs)` — 基于资源 + HP + 反读最近 N 招输出玩家 6 动作概率分布
- `scoreOutcome(myAct, mySpend, oppAct, ...)` — 调 resolveTurn 算 EV：`dmgDiff×1.4 + chargeDiff×0.6 + energyDiff×0.25 + endBonus + cliff + comboFut + tempo`
- `tempo` 推进权重：attack +1.2 / charge +0.7 / break +0.5 / power +0.4，僵局（block+block 无收益）-2.5
- `generateCandidates(mySt)` — 列举所有可行动作 + spend 档位
- `evaluateEV(myAct, mySpend, ..., dist)` — 对玩家分布加权 EV

### 三档差异

| 档 | 反读回合数 | 选择策略 |
|---|---|---|
| 新手 | 0（不反读）| 保留原概率分支 + 两条安全栏（穿透必防、spend 智能） |
| 中阶 | 3 | EV 80% top1 / 15% top2 / 5% top3 |
| 高阶 | 5 | EV 95% top1 / 5% top2 |

### 技能引导（v3.15 + v3.16 修复）
AI charged 时优先选对应动作（避免技能浪费）：
- 新手：85% 概率
- 中阶：95% 概率
- 高阶：必选

### AI 激活时机（v3.16 修复）
- useEffect 监听 phase=idle && ult.available，按难度概率 50/70/90% 决定是否激活
- 激活延迟 180-400ms（缩短自之前 700-1300ms，避免玩家快速出招打断）

## 七、底部三态决策区（v3.16 整合美术稿）

- **idle**：commit-preview 显示战术提示（多色 tactical hint，gold/cyan/risk 三级），commit-btn 灰色 disabled「选 一 个 动 作」

- tray ::before 顶部双光渐装饰（蓝紫极光）

## 八、HUD 信息层级

每方 fighter-stage 内 hud-side 列：
1. name-block（side / nameCN / archetype）
2. hud-charge-h（共鸣值 bar）

## 九、关键 Bug 修复记录

### v3.16
- **BP 屏与战斗屏敌方不一致**：`pickEnemyHero` 在两个 scene 各调用一次。修：进入 BP 时一次性 setEnemyHero 到 App state，两 scene 共享
- **演出期间可误操作**：activateUlt 加 `setPhase('ult')` 锁，commit-btn 在演出中 disabled
- **AI charged 后不会用对应动作**：aiPickAction 加 myUlt 参数，charged 时优先 ultPreferredAction
- **AI 激活时机被打断**：延迟 700-1300ms 缩到 180-400ms
- **顶部布局贴边**：brand-row padding 14→22px，enemy fighter-stage margin-top 10→16px

### v3.15
- A 烈触达初版 trigger combo≥4 + 翻倍 +3 太强且难触发：调到 combo≥3 + 仅 +2
- C 共鸣溢出初版"伤害=蓄能×2 无视防御"过强：改为穿透 + 蓄能不消耗（凝固仍能挡）

### v3.13
- 反震谐振原 streak===3 时一次性触发，避免每次都演出失去稀有感

### v3.12
- AI EV 评分让中阶/高阶在开局 78%+ 选 block，导致玩家无法建立连击 + 防御 FX 难触发。修：加 tempo 推进权重 + 僵局重罚
